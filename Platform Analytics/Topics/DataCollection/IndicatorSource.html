Indicator Sources

Indicator Sources define the sub-set of data you want to see from business tables. Data Collection executes the SQL statements defined in Conditions for every period that the job is run.

Indicator Source Example

Example: Incidents.closed indicator source with condition 'Closed on today'.

Collection retrieves the Indicator Source data for each period and uses it to calculate the Indicator Score for that period. Every Indicator Source has a dynamic DATE condition built on a Date/Time field and a time-related option.

Indicator Score Calculation

Scores are calculated by counting or aggregating records in the Indicator Source. Indicator Scores are calculated during collection using the Source (base data) and applying aggregates such as count, sum, average, min/max, as well as additional conditions.

Examples of Indicators Based on the Incidents.Closed Source
  • Number of closed incidents
  • Number of incidents closed by Self Service
  • Number of incidents not solved
  • Summed duration of closed incidents

Data Collection Terms

Data Collection enables the visualization of KPIs in dashboards by running scheduled jobs that collect scores over time. Hundreds of best practice KPIs and related collection jobs are available in bundles called Analytics Solutions or Content Packs.

A Data Collection job defines the indicators to collect and the time period for collection. By running scheduled data collection regularly, performance trends over time can be built. Indicators sharing the same source should be grouped in the same job.

Key Configuration Fields for Data Collection Jobs:

  • Indicators: Select the indicators to include in the job.
  • Relative Start / Relative End (Daily): Defines the period to collect scores for (e.g., 1 day ago for yesterdayâ€™s data).
  • Start / End Dates (Historic): Define the historical period to collect data for.
  • Frequency: Daily, Weekly, or Monthly (for periodic collection).
  • Repeat Interval (Historic): Determines how often the job runs during historical collection.
  • Active: Enable or disable the job for automatic execution.
  • Grouping: Group indicators with the same Indicator Source together.
  • Override Existing Data (Historic): Historic collection replaces existing scores for the selected period.

Data Collection Types

1. Daily (Periodic) Data Collection:

Collects indicator scores for a single day or period at regular intervals to build performance trends over time. It is the most common and smallest frequency for ongoing monitoring.

Key Configuration Fields:

  • Relative Start / Relative End - period to collect scores for (e.g., yesterday).
  • Indicators - which indicators to include.
  • Grouping - group indicators with the same source together.
  • Frequency - Daily, Weekly, or Monthly.
  • Active - enable or disable automatic execution.

2. Historic Data Collection:

Collects scores for multiple past periods to populate historical trends. Typically used in new deployments to establish baseline data.

Key Configuration Fields:

  • Start / End Dates - define the historical period to collect data for.
  • Indicators - select indicators to populate historical scores.
  • Repeat Interval - how often the job runs during historic collection.
  • Grouping - group indicators with the same source together.
  • Active - enable or disable execution.
  • Override Existing Data - replaces existing scores for the selected period.

Historic Collection Exclusions

Certain indicators cannot be accurately calculated during historic collection due to changes in record values over time. Indicators such as "Number Incidents Not Updated in Last 5 Days" or "Number Incidents Not Updated in Last 30 Days" should be excluded from historic collection and instead added to daily collection jobs to capture accurate trends.

For some breakdowns, like Assignment Group or State on open records, historical scores may not be accurate because historic collection only sees the most current value. These breakdowns should be excluded from historic collection and collected using daily jobs.

Indicator and breakdown exclusions are configured on the Data Collection Job form.
