KPI Details

KPI Details provides an in-depth analysis of Indicator performance using summary statistics, forecasting, breakdown navigation, and score comparisons. It enables a deeper understanding of business process metrics and performance trends.

Overview
  • Displays trends, breakdowns, and associated records for a specific indicator.
  • Allows users to select marker icons to explore additional information.
Trend Line and Score Summary
  • Shows the score points of the trend over time.
  • Selecting a point on the trend line updates the score summary to display the date, score, and percent change from the previous point.
  • Favorable changes (e.g., decrease in open incidents) are shown in green, and unfavorable changes are shown in red.
Date Range
  • Drill down to view score trends for a specific date range.
  • Time period can be changed by selecting two dates or using predefined periods such as 7 days, 1 month, 3 months, 6 months, YTD, or All.
Indicator Records
  • Toggle the "Show Records" button ON to view records collected for the selected time period.
  • Records are filtered based on any selected breakdown/element pairs.
  • Toggle the button OFF to return to the time series chart view.
Info and Scores
  • Select the Info icon to view indicator configurations such as description, frequency of collection, and forecast method.
  • The Scores tab shows a chronological listing of the indicator's score values.
Key Terms
  • Score Point: Each point on the trend line shows the exact score and/or target on a given day.
  • Trend Line: Approximates score movement over time; enabled by default but can be toggled off.
  • Score: The latest value of an indicator.
Navigation Options for KPI Details
  • Via Dashboards: Access KPI Details by selecting the data trend of any indicator in a dashboard.
  • Via Indicators List: Search for an indicator in Platform Analytics > Library > Indicators and select it to open KPI Details.
  • Via Indicator Form: Open an indicator in form view and use the "Explore Indicator" link to access KPI Details.
Key Definitions
  • Indicator: A business process metric used to track and optimize service performance.
  • KPI Details: Enables statistical analysis, drilldowns by attribute, and comparisons over time.
  • Data Visualization: Visualizes indicator data on dashboards using charts, trends, pies, pivot tables, and other formats.
Capabilities of KPI Details
  • Analyze indicator performance using summary statistics and forecasting.
  • Drill down by attribute or breakdown to view granular data.
  • Compare indicator scores over time to identify trends and anomalies.
  • Supports informed decision-making based on historical and current performance data.

Indicator List

The Indicator List provides a quick overview of trends and changes for all configured indicators in the system. It allows users to monitor performance at a glance and identify areas requiring attention.

Key Features of the Indicator List
  • Displays all available indicators along with their current scores.
  • Highlights trends and changes in indicator performance over time.
  • Enables filtering and searching by indicator name or attributes.
  • Provides quick access to KPI Details for deeper analysis.
  • Supports monitoring of multiple indicators from a single view.
Use Cases
  • Monitor day-to-day performance of key business processes.
  • Identify indicators that require further investigation.
  • Provide management with an overview of operational health.

Indicator Trend by Breakdown

Breakdowns are attributes used to group Indicator scores for deeper performance insights. They allow you to analyze trends based on specific categories or combinations of attributes.

Breakdown Example
  • For the Number of Open Incidents indicator, the Priority breakdown shows a separate trend for each priority value.
  • Priority breakdown elements include: 1-Critical, 2-High, 3-Moderate, 4-Low, 5-Planning.
Breakdown Navigation
  • Select the Filters button to display available breakdowns.
  • Choose a Breakdown.
  • Select a Breakdown Element to view its trend.
  • Click Apply to filter the chart by the selected element(s).
  • Trends can be navigated by a single breakdown or a combination of two breakdowns.
  • Configured breakdowns automatically appear in KPIs.
Examples of Breakdown Navigation
  • Single Breakdown: Filter scores based on a single attribute, e.g., Priority = Low.
  • Combination of Two Breakdowns: Filter scores based on two attributes, e.g., Category = Software and Priority = Low.
Important Note
  • Navigation by two breakdowns is only possible if the Collect breakdown matrix property is enabled.

Trend Visualization and Analysis

A Time Series applies different perspectives to your data so you can identify trends and patterns more easily. Aggregations combine calculations like AVG or SUM with a Period such as Week, Month, or Quarter. Platform Analytics allows you to adjust time series views dynamically after data collection.

1: Smooth the Data with a Rolling Average

  • Daily time series is the default representation of an Indicator Score trend.
  • Apply a running average (7-day, 30-day) to smooth out the trend curve.

2: Aggregate Data to a Less Frequent Period

  • Instead of daily scores, view weekly or monthly averages for a broader perspective.
  • By Period AVG aggregation recalculates trends based on the selected period.

3: Track Period to Date Scores

  • View Periodic To Date AVG (Week, Month, Quarter, Year) to understand KPI performance to date.
  • Over 50 pre-defined Time Series aggregations are available out of the box.

Visualization Types
  • Line, Area, Column, or Spline charts can be used to visualize Indicator Scores.

Analysis Tools

Use Chart Options and Statistics to better understand score trends, identify changes, and forecast future performance.

Statistics Options
  • Number of data points
  • Sum of all scores
  • Absolute and % change
  • Average, Minimum, Maximum, Median, Standard Deviation
Chart Options
  • Targets: Desired scores displayed as solid lines (black = global, grey = personal)
  • Thresholds: Horizontal dashed lines for critical levels
  • Forecast: Predict future scores based on trends
  • Trend: Linear trend line
  • Comments: Show system or user-added notes
  • Labels: Display main time series values
  • Statistics: Activate to view stats for selected date range
Time Series Options
  • Score (default)
  • Change
  • Change %
Chart Type Options
  • Line
  • Spline
  • Area
  • Column

Targets and Thresholds

  • Targets track personal or organizational goals and display a gap to target.
  • Thresholds visually indicate when scores exceed or drop below critical values.
  • Multiple thresholds can be configured; personal thresholds are grey, global thresholds are black.
  • Permissions: Admins/Power Users configure global and personal thresholds; others configure personal thresholds only.

Compare Record Sets

  • Compare records from two collection periods using the Compare button.
  • Categories in comparison include:
    • First data set: Records from the first date.
    • Moved out: Present in first but missing in second.
    • Shared: Present in both periods.
    • Moved in: Present in second but missing in first.
    • Second data set: Records from the second date.
  • Note: Compare tab is only available for Automated Indicators.

Visualization and Access Properties

KPI Details Display Options

These options are configured in the Indicator definition to control how scores are displayed in KPI Details:

  • Default time series: Sets the default time series shown in KPI Details. Exclusions can be applied.
  • Default chart type: Sets the default visualization (line, column, spline, area).
  • Live group profile: Enables the live feed icon for the selected live feed group.
  • Render continuous lines: Hides dates without scores to prevent gaps in the chart.
  • Show real-time score: Displays the current indicator score and the state of associated records.
  • Show in library: Makes the indicator visible in the Analytics Center under KPIs.

Access Control

Access to an Indicator’s score trend is governed by its Access Control settings:

  • Visible to Everyone: Default configuration; anyone can view the indicator on shared dashboards.
  • Visible to Everyone + Roles: Restricts visibility to specific roles.
  • Visible to Groups and Users: Restricts visibility to selected users or groups.

Only users with read access to the Indicator’s Facts table can view the underlying records.

Forecasting Methods

Indicators can generate forecasts using several methods:

  • Auto: System selects the best-fit method based on historical data.
  • Linear: Linear regression forecast.
  • Seasonal: Repeats the previous season’s pattern.
  • Seasonal Trend: Seasonal pattern that changes over time.
  • STL (Seasonal Trend Loess): Combines seasonal, trend, and noise filtering for best fit.
  • Random Forest: Forecasts based on decision trees.
  • Auto Regressive: Uses trend, seasonal dummies, and past values.

Period Length: Depends on Indicator frequency (Daily = 7 days, Weekly = 13 weeks, Monthly = 12 months).

Tip: Set Forecasting method to Auto for best-fit selection across all algorithms.
