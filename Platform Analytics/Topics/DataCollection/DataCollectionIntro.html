Data Collection

Data Collection is the process that enables the visualization of KPIs in Data Visualizations and Next Experience Dashboards. It provides the foundation for accurate trend analysis by capturing business process performance over time.

How Does Data Collection Work?
  • Generates periodic snapshots of business process records to build accurate process trends.
  • Retrieves Indicator Scores from process tables by running regularly scheduled jobs.
  • Jobs calculate Indicator Scores and store them in dedicated score tables.
  • Platform Analytics visualizations use these score tables to display trends in dashboards.
Four-Step Process Overview
  1. Platform Analytics data and business process data coexist within the same ServiceNow instance.
  2. Data Collection jobs run on schedule to retrieve process records.
  3. Indicator Scores are calculated and stored in Platform Analytics score tables.
  4. Dashboards and visualizations display the collected scores as trends and KPIs.
Core PA Data Tables

Indicator Scores are stored across multiple tables depending on breakdown levels:

  • Snapshots [pa_snapshots] – Stores record sys_ids.
  • Scores Level 1 [pa_scores_l1] – Stores scores for indicators and 1st-level breakdowns.
  • Scores Level 2 [pa_scores_l2] – Stores scores for 2nd-level breakdowns.
Types of Data Sources in Platform Analytics

Data for Indicators comes from several sources that drive analytics:

  1. Indicators (KPI)

    Define the measurement you want to track or calculate to assess process performance. Example: Number of open incidents.

  2. Breakdowns

    Allow grouping of indicator results by qualitative attributes such as Priority, Category, or Assignment Group. Example: Priority breakdown of open incidents.

  3. Data Collectors

    Scheduled jobs that read process tables and populate the Score and Snapshot tables. Example: [PA Incident] Daily Data Collection job.

  4. Automated Indicators

    Pre-configured indicators that are calculated automatically based on defined rules and data sources.

  5. External Data Sources

    Data imported from outside systems to enrich analytics or supplement internal process tables.

  6. Text Analytics

    Use natural language processing to extract KPIs from unstructured text data, such as incident descriptions or user feedback.

These data sources work together to ensure that Platform Analytics provides accurate, up-to-date, and actionable insights.
