Sources, Breakdowns, and Mappings

A Breakdown Source defines the elements by which you can group or analyze Indicator data. It is based on a Facts Table and optional filter conditions. Each element is usually uniquely identified by a sys_id or similar field.

Example Breakdown Source

  • Incident.Category: Breakdown elements include Inquiry/Help, Software, Hardware, Network, and Database.
  • Facts Table: The table that holds the breakdown elements.
  • Field: The unique identifier field for each element.
  • Conditions: Filters to narrow down the specific elements to include.

Types of Breakdowns

  • Automated Breakdowns: Populated automatically during data collection.
    • Tasks: Configure a Breakdown Source, create a Breakdown Mapping linking it to Indicators, enable collection.
  • Manual Breakdowns: Populated manually by importing external data.
    • Tasks: Import data manually into the breakdown table and map it to Indicators.
  • Scripted Breakdowns: Populated via custom scripts.
    • Tasks: Write a script to calculate breakdown elements dynamically, map them to Indicators, and schedule data collection.

Breakdown Mapping

Specify which field on the Indicator Source references the Breakdown Source. If no appropriate field is available, specify a script to query the Indicator Source.

  • Before you begin: Assign a Breakdown Source to the Breakdown before creating the mapping.
  • Role required: The same as for creating a Breakdown.
  • This mapping allows the Incident data to be analyzed or grouped by Breakdown values, such as Category, Priority, or State.

Manage Breakdowns Utility

  • Drag-and-drop interface to quickly add and map Breakdowns to Indicators.
  • Add new Breakdowns like Age, Priority, and State to an indicator such as Number of Open Incidents.
  • Mapped vs. unmapped Breakdowns: Adding an unmapped Breakdown automatically creates a mapping.
  • Too many Breakdown elements or combinations can impact performance. Use the Collect Breakdown Matrix property to manage multi-layer breakdowns efficiently.

Breakdown Matrix and Exclusions

  • The Breakdown Matrix allows navigation of trend data by combinations of breakdowns, e.g., Category + Priority.
  • Preview data to collect and disable unnecessary combinations using Breakdown Exclusions.
  • De-selecting cells in the Breakdown Matrix prevents data collection for specific breakdown combinations, reducing unnecessary storage and processing.

Use Cases and Benefits

  • Analyze overall trends, then drill down by Category, Priority, or Age.
  • Identify bottlenecks or areas for improvement, such as high counts in specific categories or low-priority incidents that could be automated.
  • Enable actionable insights from high-level metrics by breaking them down into meaningful segments.
