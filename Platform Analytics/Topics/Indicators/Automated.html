Automated Indicator Configuration

An Automated Indicator is based on three key elements: Indicator Source, Aggregate, and optionally Additional Conditions.

Indicator Source

The Indicator Source specifies the base table and high-level query.
Example: Incidents.Open contains all incidents that are open and unresolved to date.

Aggregate

The Aggregate defines the operation applied to the Indicator Source data. Available aggregate functions include:

  • Count: Counts all records. Example: Number of Open Incidents.
  • Distinct Count: Counts distinct values. Example: Count distinct of Resolved Incident SLAs.
  • Sum: Sums values. Example: Sum of all Reassignments of Open Incident SLAs.
  • Average: Calculates the average. Example: Average Reassignment Count of Closed Changes.
  • Minimum: Returns the minimum value in the dataset.
  • Maximum: Returns the maximum value in the dataset.

Additional Conditions

Additional Conditions refine the data from the Indicator Source.
Example: Apply the condition Reassignment count > 0 to Incidents.Open to focus only on incidents that have been reassigned.

Summary

By configuring the Indicator Source, Aggregate, and Additional Conditions, Automated Indicators automatically calculate and track metrics over time, giving actionable insights into your process performance.
