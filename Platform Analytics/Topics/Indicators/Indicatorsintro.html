Indicators Defined

An Indicator is a process metric designed to continuously measure a specific process characteristic or behavior.

Platform Analytics uses three main types of Indicators:

1. Automated Indicators

Automated Indicators are populated by scheduled Data Collection. Once configured, data trends are built automatically over time.

Example: The Number of open incidents indicator is calculated daily by counting all incidents in the Open.Incidents Indicator Source during Data Collection.

2. Formula Indicators

Formula Indicators are constructed by combining one or more existing indicators using a mathematical statement.

Example: The % of New Critical Incidents indicator is calculated by dividing the total of New Critical Incidents by the total of All New Incidents and multiplying by 100.

3. Manual Indicators

Manual Indicators are based on externally imported data. They are populated manually rather than by automated data collection.

Example: The Number of Zoom outages indicator is populated using a monthly Excel report containing outage statistics.

Sources and Indicators

Indicator Sources define the subset of data you want to see from business tables. They act as the foundation for Automated Indicators to calculate metrics.

Indicator Score Calculation

Indicator Scores are calculated by applying aggregates such as count, sum, average, min/max, and/or additional filter conditions.

Select each card to understand examples of indicators based on the Incidents.Open source:

  • Number of open incidents: Count of all records in Incidents.Open
  • Number of reassigned open incidents: Count of all records in Incidents.Open where Reassignments > 1
  • Summed age of open incidents: Sum of the age of all records in Incidents.Open

Indicator Source Configuration

An Indicator Source defines a Process Table (Facts Table) and Filter Condition(s). For every process you plan to measure, you must:

  • Identify the base process table.
  • Identify the records of interest.
  • Create an Indicator Source for those records.

For most applications, there are typically three main Indicator Sources of interest: New, Open, and Closed.

Examples of Incident Management Indicator Sources

  • NEW: Retrieves newly opened incidents for the day. Runs daily. Uses a date condition on Created or Opened fields, where the dynamic Today operator is replaced by the current date during collection.
  • OPEN: This source retrieves the cumulative number of opened and unresolved incidents on today's date and runs daily.
  • CLOSED: This source retrieves the number of closed incidents on today's date and runs daily.
  • RESOLVED: This source retrieves the number of resolved incidents on today's date and runs daily.

Automated Indicators use these Indicator Sources to calculate and track metrics over time, ensuring that the data in your KPIs reflects accurate and up-to-date process performance.
