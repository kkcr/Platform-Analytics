Dashboards and Data Visualization

Introduction

Dashboards tell a visual story about one or more business processes or services by presenting multiple widgets or data visualizations together in a single display.

Core UI Dashboards vs Next Experience Dashboards

  • Core UI dashboards are created in the classic UI using Report Designer and PA widgets.
  • Next Experience Dashboards (Platform Analytics Experience) offer a unified interface for visualizing data from multiple sources.
  • When com.glide.par.unified_analytics.enabled property is enabled, both dashboard types coexist and can be shared.

Platform Analytics Experience

  • The Platform Analytics Experience consists of two main sections:
    • Analytics Center – Displays bookmarked and certified dashboards and visualizations; includes Analytics Q/A for natural language queries.
    • Library – Stores all analytics components like dashboards, data visualizations, indicators, filters, and scheduled exports.

Dashboards in Analytics Center

  • Dashboards are created, populated, shared, and viewed in the Dashboards module.
  • Dashboards can be created both from the Analytics Center or Library sections.
  • Use the in-line editor dashboard type for most common user dashboards.

Creating a Next Experience Dashboard

  1. Navigate to All > Platform Analytics > Library > Dashboards.
  2. Select Create new dashboard action and choose In-line editor type.
  3. Enter a Name and Description (example: Name: Incident Insights, Description: Incident Operations Metrics).
  4. Select Create to open a blank dashboard canvas.

Dashboard UI and Layout Controls

  • Dashboard Layout Controls appear in Edit mode; they allow you to update the dashboard name and add tabs.
  • The Dashboard Action Menu in the upper-right corner lets you add elements to the dashboard.

Elements You Can Add to a Dashboard

  • Data Visualizations – Graphical representation of instance data or imported data.
  • Filters – Enable users to filter visualizations based on specific criteria.
  • Filter Group – Apply a set of filters to multiple visualizations simultaneously.
  • Headings – Add text at the top of a dashboard or section.
  • Image Elements – Hold static or animated images.
  • Simple Lists – Display table data customized for the dashboard audience.
  • Process Mining – Map – Shows different process states and transitions.
  • Rich Text Elements – Display text formatted as HTML or plain text, including fonts, size, and links.

This module focuses on building dashboards and data visualizations in the Platform Analytics Experience to help users analyze and share insights effectively.

Managing Dashboards

Dashboard Controls

Dashboard Controls are actions that manage the dashboard, both during and after configuration.

Working with Dashboards

  • As you make changes, periodically select Save.
  • After editing, select Exit Editing Mode to preview the dashboard.
  • Saved dashboards appear under the Owned by me filter in the Dashboards tab.
  • The Bookmark this dashboard option adds the dashboard to the Bookmarked filter for quick access.

Manage Dashboard Details

  • Select the Information (i) icon on an open dashboard to view details like Name, Description, and Visibility.
  • To make changes to details, switch the dashboard into Edit mode.

Sharing a Dashboard

  • Use the Share option in the dashboard ellipse menu to manage access.
  • Dashboards can be shared with users, groups, or specified roles.
  • Access levels include View and Edit. You can also allow users to share the dashboard with others.
  • If you have access to someone else’s dashboard, it will appear under the All filter in the Dashboards tab.

Effective management of dashboards ensures that insights are accessible to the right users while maintaining control over content and updates.
