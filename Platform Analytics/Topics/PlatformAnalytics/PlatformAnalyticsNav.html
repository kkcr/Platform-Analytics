Navigating Applications and Modules

Instance Access

The Platform Analytics Administration application contains modules that let you view and configure the different components required by business service analytics. In this section, you will access the instance and practice module navigation.

  1. Log in to the ServiceNow personal lab environment using the instance URL and administrative credentials provided by ServiceNow University.
  2. Confirm that the System Administration homepage is displayed before proceeding.
  3. For more convenience and easier navigation, select the Pin icon to pin the All menu.

Module Navigation

Type Platform Analytics in the Filter navigator field.

This action displays only those modules and applications with “Platform Analytics” in the title, including Platform Analytics Administration.
Note: All users have access to the Platform Analytics application, but not all modules such as Filters and Scheduled Export.

Performance Role-Based Analytics Navigation

Different Platform Analytics roles are restricted to specific functionality of the application. Below is a summary of what each role can do or access.

  1. PA Admin
    • Can access all modules in Platform Analytics Administration (Data Sources, Indicators, Breakdowns, External Data, Text Analytics, Data Collector, Schedules, Properties, etc.).
    • Can create Favorites for entire applications or modules.
    • Can navigate to Dashboards and Indicators across the Library.
  2. PA Power User
    • Has similar access to PA Admin for modules like Indicators, Breakdowns, Data Collector, etc.
    • Can create Favorites for applications or modules.
    • Can work with Automated Indicators and view KPI data in the Library.
  3. PA Data Collector
    • Has access to Data Collector and Data Sources.
    • Can view and manage scheduled data collection jobs.
    • Can create Favorites for Jobs and Job Logs modules.
  4. PA Contributor
    • Can access limited modules like Scoresheet.
    • Must be explicitly added to an Indicator to view its detailed score sheet.
    • Can create Favorites for Scoresheet or entire applications.
  5. PA Viewer
    • Has the most restricted access.
    • Can review contents of the Platform Analytics application but with fewer privileges compared to Contributor.
    • Can create Favorites for entire applications.

Content Pack Verification

Content Packs extend Platform Analytics with pre-built Indicators, Breakdowns, and Dashboards for specific processes or applications. Verification ensures the correct packs are installed and active in the instance.

  1. Identify Installed Content Packs
    • Search for available Content Packs in the Plugins module.
    • Review which solutions are available that align with processes or applications you manage.
  2. Confirm Installation
    • Check the Installed tab to confirm that required packs (e.g., Incident SLA Management) are present.
    • Review summary details for each installed pack.
  3. Preview New Content
    • Verify new Indicators appear in Automated Indicators within Platform Analytics Administration.
    • Use natural language filters (e.g., "indicator source contains incident") to confirm new data points.
    • Check which additional Indicators were installed with the selected Content Pack.
  4. Validate Data Collection
    • Navigate to Data Collector Jobs to ensure data collection is scheduled for the new Indicators.
    • Confirm that jobs are active and collecting analytics data as expected.

Key Insight

Navigation efficiency improves productivity. Using Favorites and search filters allows analysts and administrators to access Platform Analytics components quickly.
